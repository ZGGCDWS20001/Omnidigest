# OmniDigest 事故报告

## 事故概要

| 项目 | 内容 |
|------|------|
| 事故名称 | Dashboard 数据全为 0 - API 路径不匹配 |
| 发现时间 | 2026-03-19 |
| 解决时间 | 2026-03-19 |
| 影响范围 | Dashboard、Sources、Config 等页面数据无法显示 |
| 耗时 | 约 3 小时 |

## 问题描述

用户访问 Dashboard 页面时，所有统计数据均显示为 0。浏览器控制台无 JavaScript 错误，但 Network 请求返回 HTML 内容而非 JSON。

## 根因分析

**直接原因：**
- 前端 API 调用路径缺少 `/api` 前缀
- 例如：前端调用 `/stats/overview`，但正确路径应为 `/api/stats/overview`

**根本原因：**
1. 前端开发时未严格对照后端 API 文档
2. 后端 API Base URL 为 `/api`，但前端调用时遗漏前缀
3. Vite 代理配置正确（`/api` → 后端），但请求路径不匹配导致代理未生效
4. Vite 热更新未生效，代码修改后服务器未重启

## 影响

- Dashboard 页面数据全为 0
- Sources、Config 等页面无法正常显示
- 团队花费约 3 小时排查问题

## 解决方案

修改前端 `src/api/index.js` 中所有 API 调用，添加 `/api` 前缀。

## 改进措施

1. **前后端接口对接流程**
   - 开发前必须统一 API 文档
   - 前端按后端 API 文档实现调用路径

2. **加强 API 路径验证**
   - 使用 axios baseURL 统一配置
   - 开发阶段添加路径校验

3. **文档要求**
   - 后端必须提供完整 API 文档（含路径、参数、响应格式）
   - 前端必须按文档实现，禁止擅自简化路径
